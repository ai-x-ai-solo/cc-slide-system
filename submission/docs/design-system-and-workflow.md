# デザインシステム＆ワークフロー

---

## 1. 概要

本ドキュメントは、「本質のClaude Code 完全攻略 7dayブートキャンプ」の販売スライド自動生成システムの設計意図・AI活用方法・ワークフローを説明するものです。

- **生成ツール：** Python + python-pptx
- **AI：** Claude API（claude-opus-4-7）、ChatGPT（設計相談・壁打ち）
- **出力：** ターゲット訴求別 .pptx（Pattern A / B / C）
- **枚数：** 各パターン15〜16枚

---

## 2. 制作コンセプト

最初にデザイナーエージェントを使って自動生成しましたが、出来上がったスライドを見て「なんか違う」と感じました。

その感覚を言語化するためにChatGPTと壁打ちし、根本的な問いに辿り着きました。

**「これは読む資料か、見せる・感じさせる資料か？」**

セールスのスライドは、登壇者が話しながら見せるものです。文字で説明するのではなく、1枚で1つの感情を引き出すことが目的です。この認識から、構成・デザインを全面的に見直しました。

---

## 3. ターゲット別3パターン設計

同じサービスでも、誰に見せるかによって「刺さる言葉」「感じてほしい感情」「デザインのトーン」は変わります。

### Pattern A：副業・案件獲得訴求型
| 項目 | 内容 |
|---|---|
| ターゲット | AI副業に興味がある個人・スキルプラス受講生 |
| 感情 | 今動けば先行者になれる |
| 主軸 | 月10万円案件 / 7日 / 0.1% / 早い者勝ち |
| デザイン | 黒×パープル、未来感、数字を大きく |

### Pattern B：法人・業務改善訴求型
| 項目 | 内容 |
|---|---|
| ターゲット | 中小企業経営者・管理職・業務改善担当者 |
| 感情 | 自社の業務に使えそう |
| 主軸 | 業務効率化 / 自動化 / 人件費削減 |
| デザイン | 白×青、信頼感、整理された印象 |

### Pattern C：初心者安心訴求型
| 項目 | 内容 |
|---|---|
| ターゲット | AI初心者・PC初心者・未経験者 |
| 感情 | 自分でもできそう |
| 主軸 | 未経験OK / 日本語でOK / 質問し放題 |
| デザイン | 明るい・柔らかい・余白多め・安心感 |

## 4. スライド構成・流れの設計意図

全パターン共通の感情導線：

```
問題提起（現状への違和感）
  ↓
危機感（このままだとどうなるか）
  ↓
解決策の提示（Claude Codeという選択肢）
  ↓
信頼・権威（講師紹介）
  ↓
再現性（実績・受講生の声）
  ↓
具体的な学習内容（アクションマップ）
  ↓
価値の積み上げ（特典・パッケージ）
  ↓
価格訴求（払えそう感・ROI）
  ↓
CTA（今すぐ申し込む理由）
```

この流れはパターンによって言葉・強調点が変わりますが、感情導線の骨格は共通にしています。

---

## 5. デザイン方針

- **1スライド1メッセージ：** 1枚に伝えることは1つ。複数の主張を詰め込まない
- **1画面1感情：** 「怖さ」「期待」「安心」「決断」など、感情を1枚ずつ積み上げる
- **余白を意図的に残す：** 空白は「弱さ」ではなく「強調」のためのデザイン
- **数字を主役にする：** 7日 / 0.1% / 月10万円 / 4,500万円 など、数字は大きく・太く・目立たせる
- **登壇前提の設計：** 読まなくてもいい。見たら話し手が補足できる状態にする

## 6. 削った情報とその理由

### 講師プロフィール
| 削った情報 | 残した情報 | 理由 |
|---|---|---|
| 経歴すべて | 役割ごとに厳選4点 | 読む資料ではないため、数字と役割だけで十分 |
| みかみさんの詳細 | 「月商5億・受講生6,000名・東大」 | 権威性・規模感が伝わる3点に圧縮 |
| 佐藤さんの詳細 | 未経験→1,000万円案件→4,500万円 | 「自分にも出来そう」というストーリーラインに絞る |

### 特典・講義一覧
| 削った情報 | 残した情報 | 理由 |
|---|---|---|
| 13テーマ全リスト | 代表3テーマ | 全部見せると伝わらない。代表例で「こんなことができる」を見せる |
| 12特典すべて | 合計金額と主要3点 | 詳細より「総額249万円相当」の数字インパクトを優先 |

### 価格
| 削った情報 | 残した情報 | 理由 |
|---|---|---|
| 複数プランの比較 | 「1日あたり251円」「月々7,532円〜」 | 心理的ハードルを下げる切り口から入る |

## 7. 画像設計方針（差し替え可能構造）

本システムは「テキストと構成で意図が成立する」ことを第一に設計し、画像は後から差し替えられる構造にしています。

```
assets/references/
  profiles/
    mikami-profile.png    ← みかみさん顔写真
    sato-profile.png      ← 佐藤さん顔写真
  actionmap/
    actionmap-overview.png
    actionmap-kensakujidoka.png
    actionmap-slide-seisaku.png
    actionmap-system-kaihatsu.png
```

- ファイルが存在する場合 → 自動的にスライドに埋め込む
- ファイルが存在しない場合 → テキスト中心のプレースホルダ構成で成立
- 差し替えはファイル名を合わせて上書きするだけで完了

この設計により、写真素材が変わっても・手元に画像がなくても、システムを壊さずに運用できます。

## 8. アクションマップ紹介スライドの設計意図

13テーマすべてを一覧で見せるのではなく、代表3つに絞って「受講後に何ができるようになるか」に翻訳して見せます。

| テーマ | 翻訳した価値 |
|---|---|
| 案件探索自動化 | 営業前に案件候補が自動で集まる |
| AIエージェント×スライド制作 | 提案資料作成を半自動化 |
| はじめてのシステム開発 | AIで業務改善ツールを自分で構築 |

「何を学ぶか」より「何ができるようになるか」の言葉で見せることで、受講後のイメージを持たせます。

## 10. プロンプト連鎖・エージェント構成

```
[入力]
service_info.yaml（サービス情報）
パターン設定（ターゲット・訴求軸・配色）

    ↓

[Claude API]
各スライドのコピー・構成を JSON で生成
（1スライド1メッセージの原則に従い生成）

    ↓

[python-pptx]
JSONをもとにスライドを自動描画
色・フォント・レイアウトはパレット定義から参照

    ↓

[出力]
.pptx ファイル（パターンごとに別ファイル）
```

ChatGPTはコーディングには使わず、「どの情報を削るか」「感情導線をどう設計するか」という上流の設計相談に使いました。

---

## 11. こだわったポイント

- **削ることへの決断：** 情報を全部載せず、役割ごとに圧縮した。「削らないと伝わらない」という感覚を大切にした
- **感情導線の設計：** 単なる機能紹介ではなく、「怖さ→期待→信頼→決断」の流れを意識して構成した
- **差し替え可能な設計：** 画像がなくても成立し、追加すれば自動で使われる構造にした
- **service_info.yamlで情報を一元管理：** スライドのコピーを変えたいときはYAMLを直せばいい。コードを触らなくていい設計にした

---

## 12. 難しかった部分

- **「削る」判断：** 情報を載せすぎたくなる衝動との戦いだった。最終的にはChatGPTとの壁打ちで「1スライド1感情」という原則を軸にした
- **パターンごとの設計：** 同じサービスを違う言葉で語る難しさがあった。ターゲットの感情から逆算して言葉を選ぶことで解決した
- **Claude APIのJSON出力安定化：** レスポンスにマークダウンが混入することがあり、`[`〜`]`を正規表現で抽出するロジックを追加した
