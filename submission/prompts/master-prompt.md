# マスタープロンプト：販売スライド構成生成

---

## 使い方

以下のプロンプトをClaude APIに送信することで、ターゲット訴求別の販売スライド構成とコピーをJSON形式で生成できます。

---

## プロンプト本文

```
あなたはセールスプレゼンテーションの設計専門家です。
以下の入力情報をもとに、販売スライドの構成とコピーをJSON配列で生成してください。

【入力情報】
- サービス名: {service_name}
- ターゲット: {target}
- 訴求軸: {main_appeal}
- 引き出したい感情: {emotion}
- デザイントーン: {design_tone}
- 優先する情報: {priority_info}
- 削る情報: {remove_info}
- 使用する画像アセット: {available_images}
- 講師情報: {instructors}
- 価格情報: {price_info}
- 特典情報: {bonus_info}
- 枚数: 15〜16枚

【設計原則】
1. 1スライド1メッセージ（1枚に伝えることは1つだけ）
2. 1画面1感情（怖さ・期待・安心・決断など感情を1枚ずつ積み上げる）
3. 数字を主役にする（大きく・太く・目立たせる）
4. 登壇者が補足する前提で設計する（読まなくていい）
5. 感情導線：問題提起→危機感→解決策→信頼→再現性→価値→価格→CTA

【スライド構成】
以下の順序でスライドを設計してください。
1. カバー（タイトル・サービス名）
2. 問題提起（ターゲットの現状の課題）
3. AI格差・危機感（このままだとどうなるか）
4. 解決策提示（Claude Codeとは何か）
5. なぜ今なのか（タイミングの必然性）
6. 講師紹介①（みかみ：権威・規模感）
7. 講師紹介②（佐藤：再現性・親近感）
8. ブートキャンプ概要（7日間・形式）
9. 実際に学べる内容（アクションマップ代表3テーマ）
10. 実績・再現性（受講生の成果）
11. 受け取れる価値（特典の積み上げ）
12. 価格訴求（1日あたり・月々の払えそう感）
13. ROI（何件で元が取れるか）
14. CTA・クロージング

【画像スロット設計】
- 画像がある場合: image_path フィールドに相対パスを記載
- 画像がない場合: image_placeholder フィールドに代替テキストを記載
- どちらの場合も成立するレイアウトにすること

【出力形式】
以下のJSON配列形式で出力してください。

[
  {
    "slide_no": 1,
    "purpose": "このスライドで伝えること（1行）",
    "emotion": "引き出したい感情",
    "headline": "大見出し（短く・数字を使う）",
    "subheadline": "サブ見出し（補足）",
    "body": ["箇条書き1", "箇条書き2"],
    "image_path": "assets/references/profiles/mikami-profile.png または null",
    "image_placeholder": "画像がない場合の代替テキスト",
    "layout_hint": "レイアウトの指示（左テキスト右画像 など）",
    "design_note": "デザイン上の注意点"
  }
]

JSONのみを出力してください。説明文・マークダウン記法は不要です。
```
