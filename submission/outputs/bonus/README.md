# Bonus：Opus 4.7 × Claude Code で生成した拡張パターン

このフォルダには、本システムの基本機能（A/B/C 3パターン固定）を超えて、Claude Code（Opus 4.7）と組み合わせて生成した拡張パターンを収録しています。

## 経緯

言語化ドキュメント（`submission/docs/design-system-and-workflow.md`）の9章「削る判断」では、自由テイスト変更を「次フェーズに送った」と書きました。これは python-pptx の座標ベース設計上、Sonnet 4.6 では再現性のあるテイスト変更が難しかったためです。

ドキュメント執筆後、同じ10章「自己評価」に書いた「最大の反省点（Opus 4.7 を最初から使うべきだった）」を実際に試してみたところ、Opus 4.7 はコードを動的に書き換えて、要望通りのスライドを再現性をもって生成できました。

## 収録ファイル

| ファイル | 元パターン | 変更内容 |
|---|---|---|
| `cc_v6_AS.pptx` | A（副業） | テイストをシンプルに変更 |
| `cc_v6_BX.pptx` | B（法人） | スライド枚数を24枚に拡張 |
| `cc_v6_CS.pptx` | C（初心者） | 販売色を強めに調整 |

## このボーナスが示す本質

「AIに頼む」と「どのAIに何を頼むかを設計する」は別物——これがブートキャンプの「AIにやらせる」設計の本質です。

- **基本システム（generate_v6.py）：** ターゲット別3パターンを再現性をもって生成
- **基本システム × Opus 4.7：** 自由テイスト変更や枚数拡張も柔軟に対応

このボーナスファイル群は、その組み合わせの力を実証したものです。

詳細はLoom動画の中で実演デモを行っています。
