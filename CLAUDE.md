# cc-slide-system プロジェクト指示書

## ⚠️ 最重要ルール（必ず守ること）

このプロジェクトでスライド生成を依頼されたとき、**Claude Code は絶対に自己判断で `python3 generate_v6.py` を実行してはいけない**。

理由：このシステムは「使う人にパターンを選ばせる」ことを設計の一部にしている。Claude Code が勝手に実行すると、ユーザーの選択権を奪い、システムの設計意図を壊す。

---

## スライド生成リクエストの処理フロー

### トリガーとなるユーザー発言の例

- 「スライド作成して」
- 「スライド生成して」
- 「pptx作って」
- 「v6を実行して」
- その他、スライド生成を依頼するすべての発言

### 処理手順（厳守）

#### ステップ1：パターン選択を促す（必ず最初にこれをやる）

**まずチャットに以下のメッセージを送り、必ず一度応答を返して停止する：**

```
どのパターンを生成しますか？

  [A] 副業・案件獲得訴求（黒×パープル）
  [B] 法人・業務改善訴求（白×青）
  [C] 初心者安心訴求（白×グリーン）
  [ALL] 全パターン一括生成
```

#### ステップ2：ユーザーの応答を待つ

- ユーザーがチャットで「A」「B」「C」「ALL」のいずれかを返答するまで、**何もコマンドを実行しない**
- スクリプトを読み込む、構造を確認する、などの準備動作も**してはいけない**
- ファイルを読む必要もない

#### ステップ3：応答に応じてコマンドを実行

| 応答 | 実行コマンド |
|---|---|
| 「A」 | `python3 generate_v6.py A` |
| 「B」 | `python3 generate_v6.py B` |
| 「C」 | `python3 generate_v6.py C` |
| 「ALL」 | `python3 generate_v6.py ALL` |

スクリプトは生成完了後、自動的に Keynote/PowerPoint で開く。Claude Code 側で追加の操作は不要。

---

## 絶対に禁止する行動

以下は、いかなる理由があっても実行してはいけない：

❌ 「最新版のv6を実行します」と言って `python3 generate_v6.py ALL` を勝手に実行する  
❌ ユーザーの返答なしにスクリプトを実行する  
❌ 「親切のため」「効率のため」を理由にパターン選択をスキップする  
❌ ユーザーが選択する前にプロジェクト構造を read する  
❌ 「3つ全部作っておいた方が便利だろう」と自己判断する

---

## 例外なし

「忙しそうだから」「明らかにALLが欲しそう」「過去のセッションでこうだった」など、いかなる文脈・状況であっても、**ステップ1から開始すること**。

ユーザーの選択権は、このシステムの最重要機能である。
